# Code of Conduct

## Our pledge

We want this project to be a welcoming, supportive place for everyone who contributes — regardless of experience level, background, identity, or circumstance. We pledge to keep our interactions respectful, our feedback constructive, and our assumptions generous.

## Our standards

Contributors and maintainers are expected to:

- **Be respectful.** Disagree with ideas, not people. Critique the code or the design, not the author.
- **Assume good intent.** Most friction online comes from missing context, not malice. Ask clarifying questions before you assume the worst.
- **Be patient with newcomers.** Everyone was new once. A good answer to a beginner question helps everyone who later searches for the same thing.
- **Give credit.** If you build on someone else's idea, issue, or code, acknowledge it.
- **Keep it constructive.** If you spot a problem, explain the problem and, where possible, suggest a path forward.
- **Respect boundaries.** Not every contributor can respond on every channel or timezone. Don't escalate or ping-storm when you don't hear back quickly.

Behaviors that are not welcome in any project space:

- Personal attacks, insults, or name-calling.
- Harassment of any kind, including unwanted contact, intimidation, or persistent behavior someone has asked you to stop.
- Discrimination or exclusionary language based on identity, background, appearance, or personal circumstance.
- Publishing someone else's personal information without their consent.
- Disruptive behavior that derails productive discussion (spam, trolling, bad-faith argument).
- Conduct that a reasonable professional would consider inappropriate in a work setting.

## Scope

This Code applies to all project spaces — GitHub issues, pull requests, discussions, code review, commit messages, and any community channels (Discord, Slack, etc.) that the project formally adopts. It also applies when a person is representing the project publicly: speaking at an event as a project contributor, using an official project account, etc.

## Reporting

If you experience or witness behavior that violates this Code, please report it:

- **Private GitHub report** — open a security advisory or send a direct message to the maintainer listed in the repo.
- **Email** — contact the maintainer directly. See the project README for contact details.

All reports will be reviewed confidentially. We'll acknowledge receipt within a few days and follow up with the reporter on next steps. We will not disclose the reporter's identity without their explicit permission except where required by law.

## Enforcement

Maintainers are responsible for clarifying the standards in this Code and for taking appropriate, proportionate action in response to any behavior they judge to be inappropriate.

Typical responses, in order of severity:

1. **Private feedback.** A maintainer explains what was wrong, why, and what's expected going forward.
2. **Public correction.** A maintainer responds publicly on the thread, channel, or PR where the incident occurred.
3. **Temporary restriction.** The contributor is temporarily barred from the relevant project space.
4. **Permanent ban.** The contributor is permanently barred from all project spaces.

Maintainers who do not follow or enforce this Code in good faith may face consequences determined by project leadership.
