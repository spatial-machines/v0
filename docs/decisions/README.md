# Architecture Decision Records

Signed-off decisions that resolved contradictions between code, docs, or standards. One file per decision, numbered sequentially.

These are historical by nature — once a decision is recorded here and applied, it doesn't need to be re-litigated. New contradictions get new files.

## Format

- `NNNN-short-kebab-slug.md` — four-digit sequence number + slug
- Start with the status (RESOLVED / SUPERSEDED / DEPRECATED), the date, and the resolution in one line
- Follow with: the contradiction, the options considered, why the chosen option wins, what the resolution authorizes, what it does NOT authorize, sign-off
