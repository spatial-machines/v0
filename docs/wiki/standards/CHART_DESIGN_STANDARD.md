# Chart Design Standard

Mandatory rules for statistical charts produced by the Cartography (Visualization) agent. Applies to every chart written to `analyses/<project>/outputs/charts/`.

Charts are first-class outputs alongside maps. The same pipeline quality gates apply: a chart that overshoots evidence, misleads, or goes unlabeled is a failure.

## Families

| Family | Kinds | When to use |
|---|---|---|
| `distribution` | histogram, kde, box, violin | Shape of a single variable. Always pair with a thematic choropleth to reveal skew / outliers the map hides. |
| `comparison` | bar, grouped_bar, lollipop, dot | Rank / compare values across categories or geographies. Default report figure. |
| `relationship` | scatter, scatter_ols, hexbin, correlation_heatmap | Bivariate / multivariate relationships. Pairs with bivariate maps. |
| `timeseries` | line, area, small_multiples | Temporal change. Use small_multiples when panels > 3. |

See `config/chart_styles.json` for the authoritative family/kind registry and defaults.

## Non-negotiables

1. **Every chart must have a human-readable title.** "Poverty rate distribution" — not "poverty_rate_hist". If a caller does not supply one, the render function auto-generates a plain-language title from the field and kind. The sidecar stores the effective title, not the user arg.
2. **Every chart must carry a data-source attribution** in the footer. If the analysis is synthetic / demo, say so explicitly.
3. **Axis labels include units.** "Poverty rate (%)" not "poverty_rate". For monetary values use "Median household income (USD, 2022)". Missing units is a rework.
4. **Numeric formatting is consistent.** Thousands separators, at most 2 decimal places where meaningful. En-dash `–` for ranges, not hyphen `-`.
5. **Colorblind-safe defaults.** Categorical charts with >3 series use the `colorblind_safe` palette from `config/chart_styles.json`. Run `scripts/core/check_colorblind.py` on charts that encode category by color.
6. **Sequential palettes for sequential data; diverging for data with a meaningful midpoint.** Never use a rainbow/jet palette. Route through `charts._base.resolve_cmap_for_field()` for field-aware palette routing.
7. **Every PNG has a paired SVG and a `.style.json` sidecar.** The sidecar declares `chart_family`, `chart_kind`, palette, and effective title — this is how report-writer and publishing adapters pick charts up.
8. **No 3D.** No pie charts with more than 3 slices. No dual-axis charts. No exploded bars.

## Required chrome

| Element | Requirement |
|---|---|
| Title | Required. Plain language. 14pt bold. |
| Subtitle | Required when kind/n is non-obvious; auto-generated by default. 10pt. |
| Axis labels | Required. Include units. |
| Legend | Required when >1 series; legend title when series need disambiguation. |
| Gridlines | Light gray (`#e6e6e6`), subdued, behind data. `axis="y"` for vertical bars, `axis="x"` for horizontal. |
| Top/right spines | Hidden. |
| Attribution | Required. Lower right, 7.5pt gray. |

## Classification and color

Charts follow the same palette registry as maps. Field-name routing:

- A chart's palette is resolved from `config/map_styles.json.domain_palette_map` via `charts._base.resolve_cmap_for_field()`. The same `poverty_rate → YlOrRd` / `median_income → YlGnBu` rules apply.
- When no field match exists, fall back to the family's `default_palette` in `config/chart_styles.json`.
- Categorical charts use `palettes_categorical` from `config/chart_styles.json`. Default is `qualitative`; switch to `colorblind_safe` when encoding category by color.

## Figure size & DPI

- `distribution`: 9 × 5.5 in
- `comparison`: 10 × 6 in
- `relationship`: 8 × 7 in (square for heatmap; heatmap scales with variable count)
- `timeseries`: 11 × 5.5 in (small_multiples scales with panel count)

All at 200 DPI, white background, tight bounding box.

## Pairing rule

Whenever an analysis produces a choropleth, the Cartography agent must also produce:
- A **distribution** chart of the same field (histogram or violin) — reveals whether the choropleth color classes hide skew.
- A **comparison** chart for the top-N and bottom-N geographies — lets readers reference specific places.

Whenever an analysis produces a **bivariate** map or a **spatial regression**, also produce:
- A **scatter_ols** of the X/Y pair.
- A **correlation_heatmap** if ≥3 variables are involved.

Whenever an analysis is **change-over-time**, also produce:
- A **line** chart of the headline variable.
- **small_multiples** of per-geography trajectories when more than 4 geographies.

## Validation

Run `validate_cartography.py --charts-dir analyses/<p>/outputs/charts/` before handing off. Passing requires:

- PNG exists and is ≥ 20 KB
- `.style.json` sidecar exists, parses, and declares a valid `chart_family`
- Effective title is stored in sidecar (warning only)
- Paired `.svg` exists (warning only)

## Anti-patterns

- Rainbow / jet palettes
- Truncated y-axes that exaggerate differences
- Undeclared filter ("top 10 counties" without saying which 10 or how they were picked)
- Labels that repeat the field name instead of explaining it
- Showing precision the data does not have (e.g., 4 decimal places on an ACS estimate with MOE ±3)
- Charts that duplicate the message of the map rather than complement it
